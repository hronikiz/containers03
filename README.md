# Лабораторная работа №3: Основы контейнеризации с Docker

## Цель работы
Познакомиться с основами контейнеризации, научиться создавать Docker-образы и запускать контейнеры, а также подготовить рабочее место для последующих лабораторных работ.

## Задание
1. Установить Docker Desktop и проверить его работоспособность.  
2. Создать репозиторий `containers03` и склонировать его на компьютер.  
3. Создать `Dockerfile` и папку `site` с файлом `index.html`.  
4. Собрать образ и запустить контейнер.  
5. Проверить содержимое контейнера и удалить контейнер после проверки.

## Описание выполнения работы с ответами на вопросы

1. **Создание структуры проекта**
   - Создана папка `containers03` и подпапка `site`.  
   - В папке `site` создан файл `index.html` с произвольным содержимым:  
     ```html
     <!DOCTYPE html>
     <html>
     <head><title>Test Page</title></head>
     <body><h1>Hello, Docker!</h1></body>
     </html>
     ```

2. **Создание Dockerfile**
   - В папке `containers03` создан файл `Dockerfile` со следующим содержимым:  
     ```dockerfile
     FROM debian:latest
     COPY ./site/ /var/www/html/
     CMD ["sh", "-c", "echo hello from $HOSTNAME"]
     ```

3. **Сборка Docker-образа**
   - Команда сборки:  
     ```bash
     docker build -t containers03 .
     ```
   - **Время создания образа:** около 1–2 минут (зависит от компьютера).

4. **Запуск контейнера**
   - Команда запуска:  
     ```bash
     docker run --name containers03 containers03
     ```
   - **Вывод в консоли:**  
     <img width="581" height="49" alt="Снимок экрана 2026-03-12 020159" src="https://github.com/user-attachments/assets/974c0cea-6755-4574-a05f-3392568b2dbf" />

5. **Удаление контейнера и повторный запуск с bash**
   - Удаление контейнера:  
     ```bash
     docker rm containers03
     ```
   - Запуск с доступом к bash:  
     ```bash
     docker run -ti --name containers03 containers03 bash
     ```
   - Просмотр содержимого папки `/var/www/html/`:  
     ```bash
     cd /var/www/html/
     ls -l
     ```
   - **Вывод на экране:**  
     ```
     -rwxr-xr-x 1 root root 125 Mar 11 23:57 index.html
     ```
   - Закрытие контейнера командой `exit`.

- Файлы из проекта копируются в контейнер и доступны внутри него.  
- Работа с Docker подготовила окружение для последующих лабораторных работ.
